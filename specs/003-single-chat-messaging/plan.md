# Implementation Plan: Single Chat Messaging

**Branch**: `003-single-chat-messaging` | **Date**: 2026-05-06 | **Spec**: [spec.md](./spec.md)

## Summary

Add full messaging functionality to the existing single chat UI: real-time text messages, image/audio/file sending, and receipt — all backed by Firestore (metadata) and Supabase Storage (binary files). State management via Cubit. Clean architecture: presentation → domain → data.

## Technical Context

**Language/Version**: Dart 3 / Flutter (latest stable)
**Primary Dependencies**: flutter_bloc, cloud_firestore, supabase_flutter, record (audio), file_picker, image_picker
**Storage**: Firestore (message metadata) + Supabase Storage bucket `chatapp` (images, audio, files)
**Testing**: flutter_test / bloc_test
**Target Platform**: Android & iOS
**Project Type**: Mobile app
**Performance Goals**: Messages appear within 2 s; first 20 messages load in under 3 s
**Constraints**: No Freezed / build_runner. No Flutter imports in domain layer. Dart 3 sealed classes for state.
**Scale/Scope**: One-on-one chats; 20 messages per page

## Implementation Steps (ordered by dependency)

### Step 1 — Data Model
1. Create `lib/features/single_chat/data/models/message_model.dart`
   - `MessageType` enum: `text`, `image`, `audio`, `file`
   - `MessageModel` with all fields (see data-model.md)
   - `fromFirestore(id, data)` factory
   - `toJson()` method (Timestamp for `sentAt`)

### Step 2 — Supabase Storage Extensions
2. Add to `SupabaseStorageService`:
   - `uploadChatImage({required String chatId, required File file})`
   - `uploadChatAudio({required String chatId, required File file})`
   - `uploadChatFile({required String chatId, required File file, required String originalFileName})`
   - Paths: `chats/{chatId}/messages/images/`, `chats/{chatId}/messages/audio/`, `chats/{chatId}/messages/files/`

### Step 3 — Data Source
3. Create `lib/features/single_chat/data/datasources/messages_remote_data_source.dart`
   - Abstract: `getMessages(chatId)` → `Stream<List<MessageModel>>`
   - Abstract: `sendMessage(MessageModel)` → `Future<void>`
   - Abstract: `updateChatLastMessage(chatId, lastMessage, time)` → `Future<void>`
   - Impl uses `DataBaseService.collectionStream` for stream; `DataBaseService.setData` for writes
   - Firestore path: `chats/{chatId}/messages/{messageId}`
   - Query ordered by `sentAt` descending, limit 20

### Step 4 — Repository
4. Create `lib/features/single_chat/domain/repositories/messages_repo.dart` (abstract)
   - `getMessages(chatId)` → `Stream<ApiResult<List<MessageModel>>>`
   - `sendTextMessage(chatId, senderId, text)` → `Future<ApiResult<void>>`
   - `sendImageMessage(chatId, senderId, file)` → `Future<ApiResult<void>>`
   - `sendAudioMessage(chatId, senderId, file, duration)` → `Future<ApiResult<void>>`
   - `sendFileMessage(chatId, senderId, file, fileName)` → `Future<ApiResult<void>>`

5. Create `lib/features/single_chat/data/repositories/messages_repo_impl.dart`
   - Catch all exceptions → map to `Failure`
   - Upload via `SupabaseStorageService` then write URL to Firestore

### Step 5 — Use Cases
6. Create one use case per operation (all in `domain/use_cases/`):
   - `GetMessagesUseCase(repo).call(chatId)` → `Stream<ApiResult<List<MessageModel>>>`
   - `SendTextMessageUseCase(repo).call(chatId, senderId, text)` → `Future<ApiResult<void>>`
   - `SendImageMessageUseCase(repo).call(chatId, senderId, file)` → `Future<ApiResult<void>>`
   - `SendAudioMessageUseCase(repo).call(chatId, senderId, file, duration)` → `Future<ApiResult<void>>`
   - `SendFileMessageUseCase(repo).call(chatId, senderId, file, fileName)` → `Future<ApiResult<void>>`

### Step 6 — Cubits
7. `MessagesCubit` (sealed state: Initial/Loading/Loaded/Error)
   - `loadMessages(chatId)`: subscribes to `GetMessagesUseCase` stream
   - Cancels stream subscription on `close()`

8. `SendMessageCubit` (sealed state: Initial/Loading/Success/Error)
   - `sendText(chatId, senderId, text)`
   - `sendImage(chatId, senderId)` — calls image picker internally
   - `sendAudio(chatId, senderId, file, duration)`
   - `sendFile(chatId, senderId)` — calls file picker internally
   - Emits Loading → Success/Error; resets to Initial after short delay so input re-enables

### Step 7 — Presentation
9. `SingleChatScreen` — `StatefulWidget`
   - Receives `chatId` and `chatModel` as route args
   - Provides `MessagesCubit` + `SendMessageCubit` via `MultiBlocProvider`
   - Calls `MessagesCubit.loadMessages(chatId)` in `initState`
   - AppBar with other user's name/avatar
   - Body: `MessagesListView` + `MessageInputBar`

10. `MessagesListView` — `BlocBuilder<MessagesCubit, MessagesState>`
    - Loading: `LoadingShimmer`
    - Empty: `EmptyScreen`
    - Loaded: `ListView.builder` (reversed for latest-at-bottom); each item → `MessageBubble`

11. `MessageBubble` — `StatelessWidget`
    - `switch (message.type)` → renders text / `ImageMessageWidget` / `AudioMessageWidget` / `FileMessageWidget`
    - Sender alignment right, receiver left
    - Timestamp + sender indicator

12. `MessageInputBar` — `StatefulWidget` with `TextEditingController`, `FocusNode`
    - Text field + send button + attachment icon button
    - Send button active only when text is non-empty
    - Attachment icon opens `AttachmentBottomSheet`
    - Uses `BlocConsumer<SendMessageCubit>` for loading indicator and error toast

13. `AttachmentBottomSheet`
    - Three options: Image, Audio, File
    - Image → triggers `SendMessageCubit.sendImage`
    - Audio → shows `AudioRecorderWidget` inline
    - File → triggers `SendMessageCubit.sendFile`
    - Uses `CustomBottomSheet.showModalBottomSheetContainer`

14. `AudioRecorderWidget` — hold-to-record button
    - Uses `record` package
    - On release: uploads audio via `SendMessageCubit.sendAudio`
    - Shows recording duration counter while recording

15. `ImageMessageWidget` — shows thumbnail; taps to full-screen
16. `AudioMessageWidget` — play/pause with duration
17. `FileMessageWidget` — file name + size + download button

### Step 8 — DI Registration
18. In `injection_container.dart`:
    - `MessagesRemoteDataSourceImpl`
    - `MessagesRepoImpl`
    - All 5 use cases
    - `MessagesCubit` (factory)
    - `SendMessageCubit` (factory)

### Step 9 — Route Integration
19. Register `SingleChatScreen` in `app_routes.dart` with `chatId` + `chatModel` arguments
20. Update home chat list item tap to navigate to `SingleChatScreen`
